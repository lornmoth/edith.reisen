Github Setup

Setup Github Account

  • Create account - Github.com
  • Download Github desktop - desktop.github.com

Note, hide email in Privacy Settings:

  • Your profile > Settings > Emails: Check "Keep my email addresses private" & "Block command line pushes that expose my email"

Create a Repository

The repository is your database of website code. If you have multiple websites, or subdomains on a site, each one would have its own repository.

Steps:

  1. In the upper-right corner of any page, use the drop-down menu, and select New repository.

  2. Type the name for your site, used for your own reference. For example, "remilia" or "remilia.org".

  3. Optionally, add a description of your repository. For example, "Main remilia webpage"

  4. Click Create repository.

Put Repo Online

Github will be used to host your website. Any static site will be hosted by them at no charge.

Steps:

  1. Navigate to your repository's page
  2. Goto the settings section

  1. Scroll down to the Github Pages section near the bottom.

  2. In the source section, select the dropdown and change it from None to Main and click save

Your page will now appear online at yourusername.github.io/reponame

Create Webpage

You can create HTML/CSS pages directly in Github using its simply web text editor. It's ultimately easier to use a dedicated text-editor and upload them via Github Desktop (or Github on the command line), but we can create the first file in Github to confirm everything works.

Steps:

  1. If not in the code section of the repo, navigate to it by selecting Code in the header
  2. In the top right, click the button for Add file, then Create new file
  3. Name the file index.html
  4. Place anything in the body
  5. Under commit, you need to write a message for record purposes. Just place a "." if it's unimportant to you.
  6. Make sure Commit directly to the master branch is selected and hit Commit new file

You will then see the file in the main repo. If you click into it, you can edit the text at any time by clicking the edit button at the top right.

All history of edits for the file will also be maintained on Github's records, making it easy to go back and fix something if an update breaks it.

You should now see the page you created live when you navigate to yourusername.github.io/reponame

Introductory HTML CSS

Basic introduction to start writing webpages immediately. At the bottom of the page are more detailed guides to learn the fundamentals. HTML/CSS is very easy to pick up following them.

HTML

Basics

Tags go within <tag> and are escaped as </tag>.

Text tags:

  • <h1>, <h2>, <h3>, etc: Various level of header/title
  • <p>: main parapaghs
  • <br>: break to new line

Formatting:

  • <strong>: bold
  • <em>: italics -<quote>: quotes

Lists:

  • <ul>: unordered list, eg bullet points
  • <ol>: ordered list, eg numbers

  • <li>: list items e.g. 

          <li>Item 1</li>
      <li>Item 2</li>
  </ul>

Page tags:

  • <body>: main body
  • <nav>: navigation/links section
  • <section>: sections within the body
  • <div>: generic dividing container for css settings
  • <hr>: horizontal dividing line

Links look like this: <a href="https://url.com">Click here</a> Images look like this: <img src="img.jpg">

A basic page would like:

    <section id="introduction">
        <h1>Welcome to my Page</h1>
        <p>Welcome to my <em>page</em></p>
        <h2>Links</h2>
        <ol>
            <li>Link 1</li>
            <li>Link 2</li>
        </ol>
    </section>
    </body>

## CSS:

HTML just holds the raw content of the website. CSS holds all the styling/visual code. 

Create a file called screen.css and add this text to the HTML page, inbetween ``<head></head>``:

``<link rel="stylesheet" href="{css}">``

where {css} is the location of the css file, e.g. "screen.css"

css is written as

tag { category: setting; }



eg

body {
    color: white;
    background-color: black;
}
This will turn the main text in all the body white, and the background black; inverting default black on white. It works the same for all html tags - eg ``a { color: red;} em { color: gray;}`` Makes links red, and italicized text grayed. Settings get overriden if something is nested, e.g. if you have a section within the body then you can make the text a different color again, only for that section. Recommended template: 
html {
  font-size: 1em;
  line-height: inherit;
  margin-bottom: 2em;
}


body {
  font-family: Helvetica, Arial, sans-serif;
  color: white;
  margin: 2em;
  margin-left: 10%;
  margin-right: 10%;
  background-color: black;
  max-width: 38em;
  overflow-wrap: break-word;
}
